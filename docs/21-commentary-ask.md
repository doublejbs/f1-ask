# 21. 해설에 질문하기 — 클릭 → 상세 시트 → 대화

## 배경

해설은 세컨드 스크린 한 줄 캡션이다. 짧아야 한다. 그런데 튜닝에서 한계를 봤다 —
"왜 그 타이어인지"를 설명하면 문장이 길어지고, 같은 compound끼리 비슷해진다
(docs/20). 캡션에 모든 걸 욱여넣을 수 없다.

해법은 **층을 나누는 것**이다. 캡션은 짧게, 더 알고 싶으면 눌러서 묻는다.

> HAM 5초 페널티로 PIA에게 4위 위협  ← 캡션
> (탭) → "왜 5초야? 다른 페널티는 없었어?"  ← 질문

## 핵심 제약: 추론하지 않게 컨텍스트를 준다

Ask AI 기능은 이미 있다(`/api/ask`, `answerQuestion`). 그런데 **경기 전반 + "현재"
스냅샷**만 답한다. 과거 해설에 질문하면 지금 시점 스냅샷이 들어가 **시간이 어긋난다** —
페널티가 나온 12랩을 묻는데 44랩 순위를 보고 답한다. 환각의 원인이다.

환각 방지 규칙(`SYSTEM_RULES`: "데이터만 써라, 지어내지 마라")은 이미 강하다.
부족한 건 규칙이 아니라 **올바른 데이터를 좁혀서 주는 것**이다.

## 설계

### 시점 맥락을 해설 문서에 저장한다 (핵심 결정)

질문할 때 그 이벤트 시점의 순위를 어떻게 얻는가? `normalizeOpenF1SnapshotAt`은
`OpenF1Index`가 필요한데 ask 라우트는 접근할 수 없다(조사 확인).

**재조회하지 않는다. 해설이 이미 본 맥락을 문서에 남긴다.**

해설은 `buildCommentaryContext`로 그 시점 순위 슬라이스를 보고 생성됐다. 그 맥락을
버리지 말고 `aiCommentary` 문서에 함께 저장한다. 질문 시 그걸 컨텍스트로 쓴다.

- 재조회·재계산 없음. 라이브 OpenF1Index 접근 문제를 우회한다
- **해설이 본 것 == 질문이 보는 것.** 사용자가 캡션을 읽고 그에 대해 물으니 둘의
  맥락이 같아야 한다. 44랩이 아니라 그 이벤트의 순간을 본다
- 저장 크기: 순위 슬라이스(상위 3 + 대상 앞뒤)와 세션 상태 정도. 작다

무엇을 저장할지는 `buildCommentaryContext`가 만드는 것과 같다 — 그 함수의 출력을
문서에 직렬화하면 된다. 새 형태를 만들지 않는다.

### 클릭 진입점

`EventCommentaryLineView`는 지금 "더 보기" 확장 토글만 있다. **캡션 카드 전체를 탭하면
시트가 열린다.** 전문은 시트 안에 있으므로 인라인 확장 토글은 제거한다 — 시트가 그 역할을
대신한다. 탭 타깃이 커지고 발견성이 높다.

### 상세 시트 (신규, 기존 셸 재사용)

`BottomSheetView`(순수 셸)와 `DriverDetailSheetView`의 조립 패턴을 그대로 쓴다.
새로 만드는 건 내용과 열림 상태뿐이다.

- `RaceTabView`에 `selectedCommentary` state 추가 (`selectedDriver`와 같은 패턴)
- 시트 내용: 해설 전문 + 원본 이벤트 요약 + **그 시점 순위 맥락**(저장된 것) + 질문 입력
- footer 또는 본문에 질문 입력. 답변은 그 자리에 누적(멀티턴)

### 질문 경로 확장

`LlmQuestionRequest`·`askAiRequestSchema`·3개 provider의 `buildQuestionContext`에
**포커스 이벤트 + 그 시점 맥락**을 추가한다.

- 포커스 이벤트: 원본 `RaceEvent`(type·params·driverNumber·timestamp)
- 시점 맥락: 저장된 순위 슬라이스·세션 상태
- 프롬프트 규칙에 한 줄: "이 이벤트와 그 시점 맥락 안에서 답하라. 그 밖은 모른다고 하라"
- **3개 provider를 함께 고친다** — `buildQuestionContext`가 갈라지면 provider 교체 시
  품질이 달라진다(해설 프롬프트에서 이미 겪었다, docs/18)

포커스 컨텍스트는 optional로 추가한다. 기존 AI 탭의 경기 전반 질문은 그대로 동작한다.

### 프리필 vs 직접 호출

기존 탭투애스크는 문자열 프리필만 전달한다(구조화 컨텍스트 통로 없음). 시트에서
질문할 때는 프리필을 거치지 말고 **시트가 직접 `ask({ question, focusEvent,
focusContext })`** 를 호출한다. 문자열로 우겨넣지 않는다.

## 범위 밖

- **라이브에서 서버가 authoritative 스냅샷을 읽는 것**(ask 라우트 TODO) — 이 기능은
  저장된 시점 맥락을 쓰므로 그 TODO와 독립이다. 별건으로 남긴다
- 해설 없는 이벤트에 질문 — 이 기능은 해설 클릭이 진입점이다. 해설 없는 이벤트는
  드라이버 시트의 이벤트 이력에서 접근(기존 경로)
- 워커 본체의 이벤트별 스냅샷(docs/20 범위 밖) — 이 기능은 해설 생성 시점 맥락을
  쓰므로 무관
- Session 범위 이벤트 — 해설이 폐기됐으므로(docs/19) 진입점이 없다

## 알려진 데이터 문제 (별건)

docs/18·19에서 이어진다. 고치지 않는다. RUS `lapsCompleted: 0` 등.

## 수용 기준

1. 해설을 탭하면 상세 시트가 열린다. 기존 확장 토글과 충돌하지 않는다.
2. 시트에 해설 전문·원본 이벤트·그 시점 순위 맥락이 보인다.
3. 시트에서 질문하면 그 이벤트와 시점 맥락으로 답한다. 현재 스냅샷이 아니다.
4. 해설 문서에 시점 맥락이 저장된다. 질문 시 재조회하지 않는다.
5. 포커스 컨텍스트가 없는 기존 경기 전반 질문(AI 탭)이 그대로 동작한다.
6. 3개 provider가 같은 `buildQuestionContext`를 거친다.
7. 답변이 이벤트 범위를 벗어나면 "모른다"고 한다(환각 방지).
8. 신규 UI 텍스트 en/ko/ja. 터치 타깃 44pt.
9. 도메인 단위 테스트: 시점 맥락 저장·복원, 포커스 컨텍스트가 프롬프트에 실림.
