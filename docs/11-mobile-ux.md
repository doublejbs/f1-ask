# 11. 모바일 UX 재설계 — 하단 3탭

## 배경 · 문제

모바일(375×812) 실측 기준:

| 문제 | 측정값 |
|---|---|
| 전체 스크롤 길이 | 4,251px ≈ 5화면 분량 |
| 첫 화면 | 헤더 341px + 날씨 86px — 절반 이상이 정적 정보 |
| 순위+사이드 섹션 | 3,178px, 테이블 `min-width 860px`로 가로 스크롤 필요 |
| 이벤트 피드 | 맨 아래 (4~5번째 화면) |

세컨드 스크린의 존재 이유는 "지금 무슨 일이 일어났는지"인데, 가장 자주 변하는
정보(이벤트·배틀)가 가장 깊이 묻혀 있고, 한 번 보고 마는 정보(앱 이름, 로케일,
설명 수준, 날씨, 요약)가 첫 화면을 차지한다.

## 구조

모바일: **고정 상태바(상단) + 3탭(하단)**. 데스크톱(lg 이상): 탭 없이 3영역을
나란히 펼친다(순위 | 지금 | AI). 동일 컴포넌트를 배치만 바꾼다.

```
┌──────────────────────┐
│ 상태바 (sticky)       │  LAP 31/44 · 트랙 상태 틴트 · freshness
├──────────────────────┤
│                      │
│ 활성 탭 콘텐츠        │
│                      │
├──────────────────────┤
│  지금  |  순위  |  AI │  하단 탭바 (모바일만, pb-safe)
└──────────────────────┘
```

### 상태바 (StatusBarView)

한 줄. 모든 탭에서 sticky.

- **랩**: `LAP {currentLap}/{totalLaps}`
- **트랙 상태 틴트**: `SessionStatus` → 배경색 인코딩. 색만 봐도 판단이 끝나야 한다.
  - `Green` → 초록 틴트, `Yellow` → 노랑, `SafetyCar`/`VirtualSafetyCar` → 노랑+라벨(SC/VSC),
    `Red`/`Suspended` → 빨강, `Finished` → 중립+체커 라벨, 그 외 → 중립
- **freshness 점**: 기존 `ConnectionStatusView` 의 점만 축약(라벨은 상태 변화 시에만)
- 기존 대형 히어로(`SessionHeaderView` 341px)는 모바일에서 이 한 줄로 대체

### 「지금」 탭 (NowTabView) — 기본 진입

위에서부터:

1. **Critical 배너** — 최근 N분 내 `Critical` 이벤트(페널티·SC·적기)가 있으면 상단에
   강조 배너로. 없으면 미표시.
2. ~~배틀 위젯(BattlesView)~~ — **폐기. 배틀은 「순위」 탭 목록 인라인으로 이동했다.**
   「지금」 탭에는 배틀 섹션이 없다.
   - **근거**: 배틀은 독립된 객체가 아니라 **인접한 두 순위의 관계**다. 순위표에서
     떼어내 별도 위젯으로 두면 같은 정보(순위·드라이버·간격)를 두 번 렌더하면서
     "누가 누구 뒤에 있는가"라는 맥락은 오히려 잃는다. 순위 목록 안에서 두 행을
     시각적으로 잇는 편이 데이터의 의미에 맞고, 「지금」 탭도 이벤트 중심으로 가벼워진다.
   - 상세 규칙은 아래 「순위」 탭 항목 참고.
3. **이벤트 피드** — 기존 `EventFeedView` 재사용 (주요/전체 토글 유지)
4. **날씨 칩** — 기존 카드를 한 줄 칩으로 축약
5. **경기 요약** — `SessionStatus.Finished` 일 때만 이 탭 상단에 표시

### 「순위」 탭 (StandingsTabView)

테이블 폐기. **한 줄 = 한 명** 컴팩트 행 목록:

```
[팀컬러바] P2  LEC  ▲2   [H·23랩]  +1.9
```

- 행 구성: 포지션 · 팀컬러 바 · 코드 · 등락 · 타이어 컴파운드+수명 · 갭(선두 기준)
- 행 높이 44~48px (HIG 터치 타깃) → 20명 ≈ 1,000px ≈ 1.2화면
- 가로 스크롤 없음
- **행 탭 → 바텀 시트(DriverDetailSheetView)**: 헤드샷 · 팀명 · 섹터(퍼플/그린) ·
  최고속 · 최근 랩 · 피트 횟수 · 앞차 간격 · **"AI에게 질문" 버튼**(기존 탭투애스크가
  여기로 이사 — 시트 닫고 AI 탭으로 전환+제출)
- 즐겨찾기(★)는 목록 최상단 고정 섹션에 표시하되, 전체 목록에서도 그대로 유지한다
  (고정 섹션은 빠른 접근용 복제이고, 전체 목록의 원래 순위 맥락을 잃지 않는다).
  토글은 행의 별 아이콘 유지
- 팀 라디오 패널은 이 탭 하단에 유지
- **인라인 배틀** — 인접 순위 두 행의 간격이 좁으면 목록 안에서 직접 잇는다.
  판정은 도메인 셀렉터(`selectBattles`)를 그대로 재사용하고, 목록은 상위 N쌍이 아니라
  모든 쌍이 필요하므로 `limit` 을 드라이버 수로 준다.
  - **임계값**: `intervalToAheadSeconds` 가 1.5초 미만이면 배틀로 본다. 그중 1.0초
     미만은 매뉴얼 오버라이드 사정권으로 앰버 강조 + `OT` 칩. (1.0초는 접전 후보를
     지나치게 좁혀 실전 배틀을 놓치므로 후보 임계를 1.5초로 넓히고, 1.0초는 오버라이드
     강조선으로만 남겼다.) 리타이어·피트인이 낀 쌍은 제외한다.
  - **연결 표현**: 두 행 좌측을 잇는 2px 앰버 액센트 바(쌍의 바깥쪽 끝만 라운드) +
     아주 옅은 앰버 배경 틴트. 연속 배틀(P3↔P4↔P5)은 바가 이어져 한 덩어리로 읽힌다.
  - **수치 배치**: 간격은 뒤차의 값이므로 뒤차 행의 큰 숫자 자리를 앞차 간격(`0.6s`)이
     차지하고, 선두 갭은 그 아래 작은 줄로 내려 유지한다(순위표의 기본 정보라 버리지 않는다).
  - **접근성**: 색에만 의존하지 않도록 배틀 행에 `sr-only` 문구("앞차 {code}와 {gap}초 차
     접전")를 싣는다. `OT` 칩은 `title`/`aria-label` 로 풀네임을 유지한다.
  - 즐겨찾기 고정 섹션은 순위가 연속이 아니라 인접 관계가 성립하지 않는다. **배틀 표시를
     하지 않고 전체 목록에서만 적용한다.**
  - 배틀 상대에 대한 질문은 행 탭 → 상세 시트(앞차 간격 스탯 포함) → "AI에게 질문"으로
     흡수했다. 배틀 전용 탭투애스크는 두지 않는다.

### 「AI」 탭 (AskAiTabView)

- 기존 `AskAiView` 를 전체 화면으로. 카드 안 스크롤 제거, 입력창은 탭바 위 고정
- 탭투애스크(순위 시트·이벤트 탭)는 AI 탭으로 전환하며 질문을 제출한다
- AI 코멘터리(`AiCommentaryView`)는 이 탭 상단에 배치

### 다이어트

- 로케일 + 설명수준 스위처 → **설정 시트**(상태바 우측 ⚙ 아이콘 → 바텀 시트)
- 날씨 카드 → 「지금」 탭 한 줄 칩
- 경기 요약 → 종료 후에만
- 앱 이름/서킷명 → 상태바 좌측 축약 표기(서킷명은 설정 시트에 상세)

### 데스크톱 (lg 이상)

탭바 미표시. 3컬럼: `[순위 | 지금 | AI]`. 상태바는 상단 고정 유지.
기존 2컬럼 그리드를 대체한다.

## 상태 관리

- `DashboardTab` string enum (`Now` / `Standings` / `Ask`) — 별도 파일
- `useDashboardTabState` 훅: 활성 탭 + 탭투애스크 전환(`switchToAskWithQuestion`)
- 탭 전환 시 각 탭의 스크롤 위치는 보존하지 않아도 된다(1차 범위 밖)

## 구현 단계

1. **구조**: StatusBarView + TabBarView + 3탭 컨테이너 + 설정 시트 + 섹션 이동
2. **순위**: 컴팩트 행(DriverListView) + 상세 바텀 시트 + 탭투애스크 이동
3. **지금**: Critical 배너 + 이벤트 피드
   (배틀은 `selectBattles` 도메인 셀렉터·테스트를 유지한 채 「순위」 탭 인라인으로 이동)

## 수용 기준

1. 모바일에서 「지금」 탭 첫 화면 안에 상태바·(있다면) Critical 배너·이벤트
   피드 상단이 모두 보인다 — 스크롤 없이.
2. 순위 탭에 가로 스크롤이 없다. 20명이 1.3화면 이내에 들어간다.
3. 모든 신규 UI 텍스트가 en/ko/ja 3개 로케일로 번역된다.
4. 터치 타깃 44pt 이상(HIG), 탭바는 `pb-safe` 준수.
5. 데스크톱(lg)에서 탭바 없이 3컬럼으로 펼쳐진다.
6. 행 탭 → 시트 → "AI에게 질문" → AI 탭 전환+제출 흐름이 동작한다.
7. `selectBattles` 는 도메인 단위 테스트를 갖는다.
