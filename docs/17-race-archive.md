# 17. 지난 레이스 기록

## 배경

F1 은 대부분의 시간에 세션이 없다. 지금 앱은 세션이 없으면
`race === null` 분기로 "실시간으로 경기를 이해하세요…" 를 무한히 펄스시킨다.
**연결 중과 세션 없음을 구분하지 못해** 고장 난 것처럼 보인다.

지난 레이스 기록은 그 자체로 필요한 기능이면서, 이 빈 상태를 자연스럽게 메운다.

## 데이터 출처: Firestore 아카이브가 아니라 OpenF1 온디맨드

워커는 2026-07-21 에 배포돼 그 이후 세션만 안다. 반면 **OpenF1 은 시즌 전체를 갖고 있다**
(실측: 2026 년 완료 레이스 16 개, 과거 `session_key` 의 `session_result` 정상 조회).

따라서 아카이브 파이프라인을 만들지 않는다. 서버 라우트가 필요할 때 OpenF1 을 조회한다.

- 아카이브 코드·스토리지·정합성 관리가 없다
- 배포 즉시 시즌 전체가 생긴다
- **과거 데이터는 불변**이므로 캐시가 자연스럽다

**대신 OpenF1 자격증명이 웹 서버(Vercel)에도 필요하다.** 지금은 Cloud Functions
시크릿에만 있다. `OPENF1_USERNAME` / `OPENF1_PASSWORD` 를 Vercel 서버 전용
환경변수로 추가한다 — **`NEXT_PUBLIC_` 접두사를 붙이면 안 된다.**

## 화면

### 탭 구조: 2탭 → 3탭

`경기`(라이브) / `기록`(지난 레이스) / `AI`.

라이브 중에도 지난 레이스를 찾아볼 수 있어야 하므로 빈 상태 안에 숨기지 않고
독립 탭으로 둔다. 세션이 없을 때 `경기` 탭은 다음 세션 안내와 함께
`기록` 탭으로 유도한다.

### 목록

완료된 레이스를 최신순으로. 항목마다 라운드·그랑프리명·서킷·날짜와 **포디움 3인**.

### 상세

`buildOpenF1LiveFrame` 을 **세션 종료 시각으로 호출**해 재사용한다. 라이브와 같은
도메인 코드로 최종 스냅샷과 전체 이벤트를 얻는다 — 새 정규화 로직이 필요 없다.

표시:
- 최종 순위(포지션·드라이버·팀·갭·포인트·DNF/DSQ)
- 경기 요약(우승·포디움·패스티스트 랩·추월/피트/리타이어 수) — `RaceSummary` 재사용
- 주요 이벤트 타임라인 — 기존 이벤트 표현 재사용

## 캐시 — 이번 작업의 핵심 제약

상세 1건은 폴러가 쓰는 것과 같은 무거운 페이로드(랩·인터벌·포지션 전체)를 가져온다.
캐시가 없으면 **방문할 때마다** OpenF1 을 때린다.

- 과거 세션 데이터는 **불변**이다. 완료된 세션은 길게 캐시한다
- 목록도 캐시하되, 새 레이스가 끝나면 반영되어야 하므로 상세보다 짧게 잡는다
- 진행 중이거나 방금 끝난 세션은 캐시 대상이 아니다(`date_end` 기준으로 판정)

캐시 계층을 어디에 둘지(Next.js `revalidate` / Firestore) 판단해서 정한다.
Vercel 서버리스는 인스턴스가 자주 재생성되므로 **메모리 캐시는 신뢰하지 않는다.**

## 범위 밖

- 랩 차트·포지션 변화 그래프
- 시즌 챔피언십 순위표(드라이버·컨스트럭터 누적 포인트)
- 연도 전환(2026 만 다룬다)
- 예선·연습 세션 상세(레이스만)

## 수용 기준

1. `기록` 탭에서 2026 완료 레이스 목록이 최신순으로 보인다.
2. 항목을 열면 최종 순위·요약·주요 이벤트가 보인다.
3. OpenF1 자격증명이 클라이언트 번들에 노출되지 않는다.
4. 완료된 세션 상세가 캐시되어 반복 방문이 OpenF1 을 다시 때리지 않는다.
5. 세션이 없을 때 `경기` 탭이 로딩이 아니라 상태를 설명하고 `기록` 으로 유도한다.
6. OpenF1 조회 실패가 앱을 멈추지 않는다 — 목록·상세가 각각 오류 상태를 갖는다.
7. 신규 UI 텍스트 en/ko/ja. 터치 타깃 44pt. 팀명 잘림 0건.
