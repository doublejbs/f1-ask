# 15. 구글 로그인 + 즐겨찾기 서버 동기화

## 목적

즐겨찾기를 기기 간에 동기화한다. 지금은 `localStorage` 라 기기를 바꾸면 잃는다.

**익명 인증은 쓰지 않는다.** 익명 uid 는 기기를 바꾸면 어차피 잃어버려 동기화 목적을
달성하지 못하고, 나중에 계정 연결 시 병합 충돌 처리만 남긴다.

## 전제 (콘솔 설정)

코드만으로는 동작하지 않는다. 다음이 선행되어야 한다.

1. Firebase Auth 에서 **Google 제공업체 활성화**
2. **승인된 도메인**에 개발 접속 호스트 추가 — `localhost` 는 기본 포함이지만
   LAN IP(폰 테스트용)는 직접 추가해야 한다. 없으면 폰에서 로그인이 실패한다

## 설계

### 원칙: 로그인은 선택이다

세컨드 스크린은 TV 를 보다가 급하게 켜는 앱이다. **첫 화면에 로그인 벽을 두지 않는다.**

- 비로그인 상태로 앱의 모든 경기 데이터를 본다(경기 데이터는 공개 읽기다)
- 즐겨찾기도 비로그인으로 쓸 수 있다 — `localStorage` 유지
- 로그인은 **설정 시트**에서 선택적으로 한다. 유인은 "기기 간 동기화"다

### 저장소 전환

`useFavoriteDrivers` 가 로그인 여부에 따라 저장소를 바꾼다.

| 상태 | 저장소 |
|---|---|
| 비로그인 | `localStorage` (현행) |
| 로그인 | `users/{uid}/favoriteDrivers` (Firestore) |

Firestore 규칙은 이미 이 경로를 `isOwner(uid)` 로 보호하고 있다(`firestore.rules`).

### 로그인 시 병합

로그인 순간 두 곳에 데이터가 있을 수 있다.

- **로컬에만 있음** → 서버로 올린다
- **서버에만 있음** → 그대로 쓴다
- **양쪽에 있음** → **합집합**으로 병합한다

즐겨찾기는 순서 없는 집합이라 합집합이 안전하다. 어느 한쪽을 버리면 사용자가
"내가 별 찍은 게 사라졌다"고 느낀다. 병합 후 로컬 사본은 남겨둔다 — 로그아웃하면
다시 로컬 모드로 돌아가야 하기 때문이다.

### 로그인 방식: 팝업 vs 리다이렉트

`signInWithPopup` 을 기본으로 하되 실패 시 `signInWithRedirect` 로 폴백한다.
모바일 브라우저는 팝업을 차단하는 경우가 있고, 특히 iOS 인앱 브라우저에서 흔하다.

리다이렉트는 페이지가 다시 로드되므로 `getRedirectResult` 처리가 필요하다.

## UI

### 설정 시트

기존 `SettingsSheetView` 에 계정 섹션을 추가한다.

- 비로그인: "Google 로 로그인" 버튼 + 한 줄 설명("즐겨찾기를 기기 간에 동기화합니다")
- 로그인: 표시 이름 · 이메일 · 프로필 사진 + "로그아웃"

로그인 상태를 상태바에 노출하지 않는다 — 상단은 경기 정보에 집중한다.

### 동기화 상태

로그인 직후 병합이 끝나기 전까지 즐겨찾기가 잠깐 비어 보일 수 있다.
낙관적으로 로컬 값을 먼저 보여주고 서버 값이 도착하면 합친다.

## 실패 처리

인증·동기화 실패가 **앱을 멈추면 안 된다.** 경기 데이터는 인증과 무관하다.

- 로그인 실패 → 설정 시트에 오류 문구, 앱은 계속 동작(비로그인 모드)
- Firestore 즐겨찾기 읽기/쓰기 실패 → `localStorage` 로 폴백, 조용히 로그만
- 팝업 차단 → 리다이렉트로 자동 폴백

## 범위 밖

- 이메일/비밀번호 로그인, 애플 로그인
- 계정 삭제 플로우(규칙상 `users/{uid}` delete 는 이미 서버 전용으로 막혀 있다)
- 알림 설정 동기화(`notificationSettings` 경로는 준비돼 있으나 기능이 없다)

## 수용 기준

1. 비로그인 상태로 앱의 모든 경기 데이터와 즐겨찾기가 동작한다.
2. 설정 시트에서 구글 로그인·로그아웃이 된다.
3. 로그인 시 로컬·서버 즐겨찾기가 합집합으로 병합된다.
4. 로그인 상태에서 즐겨찾기를 바꾸면 Firestore 에 반영되고, 다른 기기에서 보인다.
5. 로그아웃하면 로컬 모드로 돌아가며 앱이 멈추지 않는다.
6. 팝업이 차단되면 리다이렉트로 폴백한다.
7. 인증·동기화 실패가 경기 데이터 표시를 막지 않는다.
