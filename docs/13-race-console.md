# 13. 레이스 콘솔 — 순위·이벤트·해설을 한 화면에

## 문제

경기를 볼 때 알고 싶은 것은 세 가지다.

| 질문 | 현재 위치 |
|---|---|
| 누가 몇 위인가 | 「순위」 탭 |
| 방금 무슨 일이 있었나 | 「지금」 탭 |
| 그게 무슨 의미인가 | 「AI」 탭 |

세 가지가 서로를 설명하는 관계인데 탭 세 개로 갈라져 있다. "HAM에게 5초 페널티"를
보고 순위 영향을 확인하려면 탭을 옮겨야 하고, 돌아오면 피드 위치를 잃는다.

### 더 큰 문제: 이벤트 피드와 AI 해설이 같은 데이터다

`isCommentaryEligible`([AiCommentary.ts](../packages/domain/src/ai/AiCommentary.ts))과
`PRIMARY_EVENT_PRIORITIES`([PrimaryEventPriorities.ts](../packages/domain/src/PrimaryEventPriorities.ts))의
선별 기준이 **완전히 동일**하다 — 둘 다 `High | Critical`.

게다가 `AiCommentary` 는 `sourceEventId` 를 갖는다. 즉 해설은 이벤트에
**1:1로 종속된 파생 데이터**인데, 지금은 서로 다른 탭에서 독립된 목록 두 개로
렌더된다. 사용자는 같은 사건을 두 번 읽는다.

## 설계

### 원칙 1: 해설은 이벤트의 한 겹이지 별도 목록이 아니다

이벤트 피드와 AI 해설을 **하나의 항목**으로 합친다.

```
● HAM에게 5초 페널티가 부과되었습니다 (충돌 유발).        22:31
  └ 3랩 남은 시점의 5초 페널티는 사실상 순위 방어가
    불가능합니다. 뒤차와 간격이 2.1초라 최종 순위가 뒤집힙니다.
```

- 윗줄: 도메인이 만든 결정론적 문장(`translateRaceEvent`) — 항상 있다
- 아랫줄: LLM 해설 — 있을 때만, 없으면 윗줄만
- `AiCommentaryView` 와 별도 해설 목록은 제거한다

이 구조의 이점: LLM 이 실패(크레딧 소진·타임아웃)해도 **이벤트 문장은 남는다.**
현재는 해설 목록 전체가 목 텍스트로 조용히 바뀌어 사용자가 구분할 수 없다.

### 원칙 2: 순위와 이벤트를 동시에 본다

모바일에서 **순위(뒤) + 이벤트 시트(앞)** 분할 레이아웃을 쓴다. iOS 지도의
바텀 시트 패턴이다.

```
┌──────────────────────┐
│ 상태바                │
├──────────────────────┤
│ 01 ANT    Mercedes   │
│ 02 LEC ┐  Ferrari    │  ← 순위 (스크롤)
│ 03 VER ┘  Red Bull   │
│ 04 PIA    McLaren    │
├══════════════════════┤  ← 드래그 핸들
│ ● HAM 5초 페널티      │  ← 이벤트+해설 (스냅 3단)
│   └ AI 해설…          │
│ ● BOR가 NOR를 추월     │
└──────────────────────┘
```

**스냅 포인트 3단**
| 단계 | 높이 | 용도 |
|---|---|---|
| 접힘 | 핸들 + 최신 1건 | 순위 위주로 볼 때 |
| 기본 | 화면 45% | 균형 (기본값) |
| 펼침 | 화면 85% | 이벤트 몰아볼 때 |

시트가 접혀 있어도 **최신 이벤트 1줄은 항상 보인다** — 놓치지 않기 위함이다.
`Critical` 이벤트가 들어오면 시트를 기본 단계로 자동 확장한다(1회, 사용자가
접은 뒤에는 재확장하지 않는다).

### 원칙 3: 순위 행을 탭하면 그 드라이버로 이벤트를 좁힌다

지금은 행 탭 → 상세 시트다. 여기에 **드라이버 필터**를 더한다.

- 행 탭 → 상세 시트(기존 유지)
- 상세 시트에 "이 드라이버 이벤트만 보기" → 시트를 닫고 이벤트 시트를 해당
  드라이버로 필터 + 필터 칩 표시
- 필터 칩의 X 로 해제

이벤트의 `driverNumber` / `params.driverCode` / `params.driverCodes` 로 매칭한다.
(`driverCodes` 는 다중 차량 인시던트를 쉼표로 담는다 — 부분 일치가 필요하다.)

## 탭 구조 변경

「지금」과 「순위」가 하나로 합쳐지므로 **3탭 → 2탭**이 된다.

| 기존 | 변경 |
|---|---|
| 지금 / 순위 / AI | **경기** / **AI** |

- **「경기」**: 상태바 + 순위 + 이벤트 시트. Critical 배너와 날씨 칩은 순위 상단에.
- **「AI」**: Ask AI 대화 전용. 해설 목록이 빠지고 순수 대화 화면이 된다.

「지금」 탭에 있던 요소의 행선지:
- Critical 배너 → 「경기」 순위 영역 상단(sticky)
- 날씨 칩 → 상태바 옆 또는 순위 상단
- 경기 요약(Finished) → 「경기」 최상단
- 이벤트 피드 → 이벤트 시트

## 데스크톱

이미 3컬럼이라 시트가 필요 없다. `[순위 | 이벤트+해설 | AI]` 로 유지하되,
가운데 컬럼이 **합쳐진 피드**가 된다. 드라이버 필터는 동일하게 동작한다.

## 범위 밖

- 시트 드래그 제스처의 물리 기반 애니메이션(스냅 전환은 CSS transition 으로 충분)
- 이벤트 검색·날짜 점프
- 해설 재생성 버튼

## 수용 기준

1. 이벤트 항목 하나에 도메인 문장 + (있으면) AI 해설이 함께 렌더된다.
   별도 AI 해설 목록이 존재하지 않는다.
2. LLM 이 실패해도 이벤트 문장은 정상 표시된다.
3. 모바일에서 순위와 이벤트가 스크롤 없이 동시에 보인다(기본 스냅).
4. 시트가 접혀도 최신 이벤트 1건이 보인다.
5. `Critical` 이벤트 수신 시 시트가 기본 단계로 1회 자동 확장된다.
6. 순위 상세 시트에서 드라이버 필터를 걸면 이벤트가 좁혀지고 칩으로 해제할 수 있다.
7. 탭이 「경기」/「AI」 2개다. 기존 탭투애스크·즐겨찾기·팀라디오·배틀 인라인 회귀 없음.
8. 신규 UI 텍스트 en/ko/ja 3로케일. 터치 타깃 44pt. 시트 접근성(`role="dialog"`
   는 모달이 아니므로 부적절 — 논모달 시트로 `aria-expanded` 기반 처리).
