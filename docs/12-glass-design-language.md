# 12. 디자인 언어 — 앰비언트 워시 + 애플 글래스모피즘

레퍼런스 앱(사용자 제공 스크린샷 5장) 기준으로 디자인 언어를 재정의한다.

## 레퍼런스에서 읽어낸 원칙

### 1. 순수 검정 베이스 + 컨텍스트 앰비언트 워시

배경은 네이비(`225 40% 6%`)가 아니라 **순수 검정**이다. 그 위에 화면 상단에서
큰 반경의 컬러 워시가 깔리고 아래로 갈수록 검정으로 사라진다. 워시 색은
**컨텍스트에서 나온다** — 메르세데스 선두면 청록, 페라리 상세면 빨강,
실버스톤이면 파랑.

현재 우리 배경은 고정된 3색 라디얼(빨강·파랑·보라)이 항상 같은 자리에 있다.
이것을 **선두 드라이버 팀 컬러 기반 동적 워시**로 바꾼다.

- 워시는 화면 상단 60~70%에 하나의 큰 라디얼로만. 여러 개 겹치지 않는다.
- 세션이 적기/SC면 팀 컬러 대신 해당 경보색이 워시를 지배한다(상태가 색을 이긴다).
- 워시는 `background-attachment: fixed` 로 스크롤과 무관하게 고정.

### 2. 카드가 아니라 행

레퍼런스에는 **리스트를 감싸는 카드가 없다.** 행과 행 사이 헤어라인 구분선만
있고, 배경은 앰비언트 워시가 그대로 비친다. 유리 패널은 **떠 있는 것**
(탭바, 시트, 버튼)에만 쓴다.

우리는 현재 모든 섹션이 `glass-panel` 카드다. 이것을 두 종류로 가른다.

| 종류 | 용도 | 스타일 |
|---|---|---|
| **플레인 섹션** | 순위 목록, 이벤트 피드, 스탯 목록 | 배경 없음. 섹션 제목 + 헤어라인 구분선 행 |
| **글래스 표면** | 탭바, 바텀 시트, pill 버튼, 배너 | 반투명 + 블러 + 상단 하이라이트 |

### 3. 큰 숫자, 작은 라벨

핵심 수치는 압도적으로 크다(`204 PTS`, `02 POS`, `5.891 KM`). 라벨은 그 옆이나
아래에 작게 붙는다. 아이콘 + 큰 숫자 + 작은 라벨의 스탯 행이 반복 패턴이다.

- 히어로 수치: 44~56px, `font-weight 700`, `tracking-tight`
- 단위/라벨: 12~13px, `text-muted-foreground`
- 순위 번호는 **0 패딩 2자리**(`01`, `02`) + 뮤트 색

### 4. 팀 컬러는 텍스트 색으로

레퍼런스는 팀명을 팀 컬러 텍스트로 쓴다(`Mercedes` 청록, `페라리` 빨강).
우리는 현재 세로 바로만 쓰는데, **바 + 팀명 텍스트 색**을 함께 쓴다.

### 5. 떠 있는 탭바

탭바가 화면 바닥에 붙어 있지 않다. **좌우 여백을 두고 떠 있는 알약**이며,
콘텐츠가 그 아래로 흘러 비친다. 활성 탭은 더 밝은 라운드 사각 칩.

현재 우리 탭바는 바닥에 꽉 찬 사각형(`rounded-none border-x-0`)이다.

### 6. 원형 글래스 아이콘 버튼 · 알약 액션 버튼

- 뒤로가기/닫기: 반투명 원형
- 보조 액션: 아이콘 + 라벨의 반투명 알약(`타임라인`, `결과`, `서킷 통계`)

## 토큰

`globals.css` 를 다음으로 개정한다. **모든 CSS 값은 한 줄로 작성한다** —
이 레포의 css-loader 는 중첩 함수를 포함한 멀티라인 값을 파싱하지 못한다.

```
--background: 0 0% 0%;          /* 순수 검정 */
--surface-glass: 0 0% 100% / 0.06;
--hairline: 0 0% 100% / 0.08;
--ambient-hue: <런타임 주입>;    /* 선두 팀 컬러 */
```

앰비언트 워시는 `<body>` 가 아니라 대시보드 루트에 인라인 스타일로 주입한다
(팀 컬러가 런타임 값이므로). `--ambient-color` CSS 변수를 세팅하고 CSS 는
그 변수를 참조한다.

### 글래스 계층

| 클래스 | 용도 | 블러 | 배경 |
|---|---|---|---|
| `.glass-float` | 탭바·플로팅 버튼 | 24px | `white/0.07` |
| `.glass-sheet` | 바텀 시트 | 32px | `white/0.09` |
| `.glass-chip` | pill 버튼·배지 | 16px | `white/0.06` |

모두 상단 1px 하이라이트(`inset 0 1px 0 white/0.10`) + 헤어라인 보더.

### 구분선

`.hairline` — `border-bottom: 1px solid hsla(0,0%,100%,0.08)`. 마지막 행은 없음.

## 화면별 적용

### 상태바
글래스 유지하되 알약 형태로 좌우 여백. 랩 숫자를 크게(`text-2xl`), 트랙 상태는
컬러 점 + 라벨.

### 「지금」 탭
- Critical 배너: 글래스 + 좌측 컬러 액센트 바 유지, 라운드 상향
- 배틀: 카드 제거 → 섹션 제목 + 헤어라인 행. 갭 숫자를 크게
- 이벤트 피드: 카드 제거 → 섹션 제목 + 헤어라인 행. 우선순위 배지는 작은 점/칩으로 축소
- 날씨: 글래스 칩 유지

### 「순위」 탭
카드 제거. 행 구조를 레퍼런스에 맞춘다.

```
01   ANT              선두    ›
     Mercedes   S·3랩
```

- 좌: 0패딩 2자리 순위(뮤트, `text-lg`) + 팀컬러 세로 바
- 중: 코드(`text-lg font-bold`) / 팀명(팀컬러, `text-xs`) + 타이어
- 우: 갭(`text-lg font-bold tabular-nums`) + 등락 + 시���론
- 행 높이 56px+ (레퍼런스가 우리보다 넉넉하다)

### 「AI」 탭
말풍선은 글래스 칩. 입력창은 떠 있는 알약.

### 바텀 시트
`glass-sheet` + 상단 그랩 핸들. 드라이버 상세는 헤드샷을 크게 두고 아래에
아이콘+큰숫자+라벨 스탯 행을 헤어라인으로 나열(레퍼런스 IMG_4676 패턴).

## 수용 기준

1. 배경이 순수 검정이고, 선두 드라이버 팀 컬러 기반 앰비언트 워시가 상단에 깔린다.
   세션이 적기/SC면 경보색이 워시를 지배한다.
2. 순위·이벤트·배틀 목록에 카드 래퍼가 없다. 헤어라인 구분선만 있다.
3. 탭바가 바닥에 붙지 않고 좌우 여백을 둔 떠 있는 알약이다.
4. 순위 번호가 0 패딩 2자리이고 팀명이 팀 컬러 텍스트다.
5. 핵심 수치(랩·갭·포인트)가 이전보다 확연히 큰 타이포다.
6. 기존 기능 회귀 없음 — 탭 전환, 시트, 탭투애스크, 필터 토글 모두 동작.
7. 접근성 유지 — 44pt 터치 타깃, `role="dialog"`, `aria-current`, 대비.
