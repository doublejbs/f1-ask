# 24. 순위표 중심 라이브 — 예측 행 인라인

## 배경

벨기에 GP 리플레이를 실제로 보며 내린 결정. 라이브 화면에서 순위표가 본체인데
상단 영역(세션 상태 + 최근 이벤트 페이저)과 카드가 순위표를 밀어낸다. 추월
예측(docs/23)은 "다음에 볼 것"이라 순위표의 해당 행에 붙는 게 가장 자연스럽다 —
배틀 인라인 표시(docs/13 §행 인라인)와 같은 원리다.

사용자 결정:

- 예측은 **순위표 행 인라인** — 쫓는 드라이버(chaser) 행에 "3랩 후 LAW" 배지
- 다른 지금 볼 것 신호(타이어 한계·피트 직후·순위 급변·간격 접근)는 카드 **유지**
- 상단 이벤트 영역은 **세션 상태만 남김** — 최근 이벤트 페이저 제거 (이벤트는
  피드 탭에서 본다)

## 설계

### 행 인라인 배지

- 데이터: 스냅샷의 `overtakeForecasts` (워커가 계산 — 클라이언트 계산 없음)
- 도메인 셀렉터: `selectOvertakeForecastsByChaser(snapshot)` →
  `Map<driverNumber, OvertakeForecast>`. 행 렌더가 O(1) 로 찾는다
  (markersByDriver 패턴)
- 표시: chaser 행에 "{N}랩 후 {target}" 배지. 배틀 인라인 칩과 같은 시각 언어,
  단 배틀(현재 접전)과 구분되는 예측 톤(점선/보조색 등 기존 디자인 토큰 내에서)
- target 행에는 표시하지 않는다 — 한 예측이 두 행에 붙으면 중복 소음
- 같은 행에 배틀 칩과 예측 배지가 동시에 성립하면 **배틀이 이긴다** (현재가 예측보다
  우선). 예측은 배틀 부재 시에만 그린다
- i18n: ko "{laps}랩 후 {target}" / en "{target} in {laps} laps" (1랩 단수) / ja

### 상단 정리

- `RaceTabView` 에서 `LatestEventPagerView` 제거. `SessionStatusStripView` 는 유지
  (SC·레드 플래그는 안전 정보)
- `WatchNowLanesView`·`DriverListView` 는 그대로

### 지금 볼 것 카드의 예측 신호

카드 유지 결정에 따라 forecast 신호도 카드에 남긴다(#50). 행 인라인과 중복되지만
칸 배치(내 드라이버)와 다양성 캡이 있어 소음이 아니며, 실사용 후 카드 쪽 제거를
재검토한다.

## 범위 밖

- LatestEventPagerView 컴포넌트 삭제 (다른 화면에서 쓸 수 있어 사용처만 제거)
- 카드 쪽 forecast 신호 제거 여부 — 실사용 후

## 수용 기준

1. chaser 행에 예측 배지가 인라인으로 뜬다. target 행에는 안 뜬다
2. 같은 행에서 배틀 칩이 있으면 예측 배지는 숨는다
3. 상단에서 최근 이벤트 페이저가 사라지고 세션 상태 스트립만 남는다
4. 도메인 셀렉터 단위 테스트 (빈/복수/중복 chaser 방어)
5. ko/en/ja 번역, 1랩 단수 처리
